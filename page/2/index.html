HashMap与几种类似结构比较

发表于 2017-07-24 | 分类于 基础 ， 数据结构 | 阅读次数:

HashMap

HashMap在1.7以及1.7以前，都用的是数组加链表的结构来存储的，通过Hash算法，来判断数据在哪个数据里面，然后添加到链表末端。
1.8之后，HashMap在数据量达到一定数值后，会形成数组+链表+红黑树的结构。
但是HashMap一直是线程不安全的，在多线程put的情况下，get的时候会出现死循环，而不是死锁。在扩容的时候也会发生。
多线程下put的时候也可能导致元素丢失。

动态代理

所谓代理，就是对目标对象的方法调用是通过代理对象来完成的。
代理类在程序运行时创建的代理方式被成为动态代理。
动态代理创建对象的过程如下：

  • 通过实现InvocationHandler接口创建自己的调用处理器 IvocationHandler handler = new InvocationHandlerImpl(…);
  • 通过为Proxy类指定ClassLoader对象和一组interface创建动态代理类
    Class clazz = Proxy.getProxyClass(classLoader,new Class[]{…});
  • 通过反射机制获取动态代理类的构造函数，其参数类型是调用处理器接口类型
    Constructor constructor = clazz.getConstructor(new Class[]{InvocationHandler.class});
  • 通过构造函数创建代理类实例，此时需将调用处理器对象作为参数被传入
    Interface Proxy = (Interface)constructor.newInstance(new Object[] (handler));

同步阻塞IO（Java BIO）

同步并阻塞，服务器实现模式为一个连接一个线程，即客户端有连接请求时服务器端就需要启动一个线程进行处理，如果这个连接不做任何事情会造成不必要的线程开销，当然可以通过线程池机制改善。
BIO比较适用于连接数小且固定的架构，这种方式对服务器资源要求比较高，并发局限于应用中。

ConcurrentHashMap源码解读笔记

发表于 2017-07-19 | 分类于 基础 ， 数据结构 | 阅读次数:

参考文档
ConcurrentHashMap是concurrent家族中的一个类，由于它可以高效地支持并发操作，以及被广泛使用，经典的开源框架spring的底层数据结构就是使用ConcurrentHashMap实现的。跟HashTable和synchronizeMap比，后两者在方法或者参数上添加了synchronized锁，而concurrentHashMap摒弃了Segment（锁段）的概念（在1.8之前的结构还是按照锁段结构），而是启用了一种全新的方式实现,利用CAS算法。所以，相较性能会有优势很多，适用于读多写少的场景（因为concurrentHashMap对写加锁，对读不加锁）。而且concurrentHashMap底层沿用了HashMap中1.8里面数组+链表+红黑树的方式思想。
