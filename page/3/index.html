java的四种引用方式以及使用场景

发表于 2017-07-17 | 分类于 基础 ， 概念 | 阅读次数:

参考文档
java的四种引用方式包括：强引用，软引用，弱引用，虚引用

使用这几种引用方式主要目的是为了：
1.通过代码的方式决定某些对象的生命周期。
2.有利于jvm进行垃圾回收。

基本概念

强引用

强引用是指创建一个对象并把这个对象赋给一个引用变量。
比如：
Object object =new Object();
String str =”hello”;
强引用有引用变量指向时永远不会被垃圾回收，jvm会抛出OOM也不会回收对象。
如果想中断强引用和某个对象之间的关联，可以显示地将引用赋值为null，这样一来的话，JVM在合适的时间就会回收该对象。

类加载机制

发表于 2017-07-17 | 分类于 基础 ， JVM | 阅读次数:

关于类加载器所做的事情，已经在前一篇文章做了介绍。此处先介绍类加载器体系结构

如图所示，类加载器可分为几类：

（1）启动类加载器

负责加载存放在%JAVA_HOME%\lib目录中的类库，并且被java虚拟机识别的类库（识别方式是按照文件名，所以，名字不符的就算放入也不会加载），启动类加载器无法被java程序直接引用。

（2）扩展类加载器

负责加载%JAVA_HOME%\lib\ext目录中的类库。

（3）应用程序类加载器

负责加载用户类路径classpath上所指定的类库

（4）自定义类加载器

如果程序中没有自定义过类加载器，应用程序类加载器就是程序中默认的类加载器。
若要实现自定义类加载器，只需要继承java.lang.ClassLoader 类，并且重写其findClass()方法即可。

GC类型

目前流行的GC类型有五种，分别是：

  • Serial GC (-XX:+UseSerialGC)
  • Parallel GC (-XX:+UseParallelGC)
  • Parallel Old GC (Parallel Compacting GC) (-XX:+UseParallelOldGC)
  • Concurrent Mark & Sweep GC (or “CMS”) (-XX:+UseConcMarkSweepGC)
  • Garbage First (G1) GC

此处只介绍我关心的两种GC类型：CMS与G1

CMS

“Concurrent Mark and Sweep” 是CMS的全称，官方给予的名称是：“Mostly Concurrent Mark and Sweep Garbage Collector”;
年轻代：采用 stop-the-world mark-copy 算法；

年老代：采用 Mostly Concurrent mark-sweep 算法；

JVM分区以及垃圾回收

发表于 2017-07-11 | 分类于 基础 ， JVM | 阅读次数:

Java虚拟机垃圾收集器关注的内存结构如下


如图所示：堆内存被分为新生代和老年代两个部分，整个堆内存使用分代复制垃圾收集算法。方法区：一般被称为永久代。
其中详细的算法会在垃圾回收算法详解中进行描述。

新生代

新生代使用的是复制和标记-清除垃圾算法。
新生代中98%的对象是朝生夕死的短生命周期对象。所以不需要将新生代分为容量大小相等的两部分内存。
如果空白的survivor空间无法存放下仍然存活的对象，使用内存分配担保机制，直接将新生代依然存活的对象复制到年老代内存中，同时对于创建大对象时，如果新生代中无足够的连续内存时，也直接在年老代中分配内存空间。
Java虚拟机对新生代的垃圾回收称为Minor GC，次数比较频繁，每次回收时间也比较短。
使用java虚拟机-Xmn参数可以指定新生代内存大小。
