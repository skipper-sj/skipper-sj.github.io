动态代理以及反射基本概念

发表于 2017-07-24 | 分类于 基础 ， 动态代理 | 阅读次数:

动态代理

所谓代理，就是对目标对象的方法调用是通过代理对象来完成的。
代理类在程序运行时创建的代理方式被成为动态代理。
动态代理创建对象的过程如下：

  • 通过实现InvocationHandler接口创建自己的调用处理器 IvocationHandler handler = new InvocationHandlerImpl(…);
  • 通过为Proxy类指定ClassLoader对象和一组interface创建动态代理类
    Class clazz = Proxy.getProxyClass(classLoader,new Class[]{…});
  • 通过反射机制获取动态代理类的构造函数，其参数类型是调用处理器接口类型
    Constructor constructor = clazz.getConstructor(new Class[]{InvocationHandler.class});
  • 通过构造函数创建代理类实例，此时需将调用处理器对象作为参数被传入
    Interface Proxy = (Interface)constructor.newInstance(new Object[] (handler)); 动态代理必须有一个共同的父类Proxy。Java的继承机制注定了这些动态代理类无法实现对class的动态代理，只能实现接口的代理。
    后续会补充动态代理类的一些内容。

    反射

    所谓反射，就是在运行状态中，对于任意一个类，都能够知道这个类的所有属性和方法；对于任意一个对象，都能够调用它的任意一个方法和属性；这种动态获取的信息以及动态调用对象的方法的功能称为java语言的反射机制。
    反射机制能够让我们在运行时，判断任意一个对象所属的类、构造任意一个类的对象、判断任意一个类所具有的成员变量和方法、调用任意一个对象的方法、生成动态代理。
